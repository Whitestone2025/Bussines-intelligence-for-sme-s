# Mexico Business Quality

- Prefer MXN unless the business clearly sells in USD.
- Prefer Mexico as default geography unless the founder says otherwise.
- Treat WhatsApp, referrals, Instagram, Facebook, and Google Maps as first-class channels when relevant.
- Reward visible proof, transparent process, and specific deliverables.
- Penalize vague imported startup language that sounds disconnected from Mexican small-business reality.
- Surface payment clarity, invoicing confidence, and local trust as practical conversion factors.

## Consulting Logic Standard

- Start with an explicit problem statement before analysis. The system should define what decision must be made, for whom, under what constraints.
- Use a hypothesis-led structure. Every major analysis should make the current working thesis visible instead of listing disconnected facts.
- Build a robust fact base. Major claims should cite evidence, benchmarks, or assumptions explicitly; if the fact base is thin, the output must say so.
- Separate `evidence`, `assumption`, `inference`, `validated_fact`, and `decision`. Do not blur these states inside recommendations.
- Test the recommendation against execution reality. Recommendations should reflect actual delivery capacity, channel access, founder constraints, and implementation risk.
- Include contrary evidence when it exists. The system should not only stack support for the preferred thesis.
- Show options, not only a single answer. For meaningful decisions, present at least a primary recommendation and one credible alternative path.
- Make recommendation criteria transparent. The output should show why one option wins: economics, speed, risk, trust, channel fit, or capability fit.
- Assess competitor reaction and substitution risk when relevant, not only market size or customer pain.
- Prefer rapid learning loops. Early moves should create quick wins, practical signal, or funding for later moves.

## Deliverable Standard

- Every executive deliverable should contain: decision question, current thesis, supporting fact base, key assumptions, recommendation, risks, and immediate next actions.
- Every diagnosis should explain what is known, what is inferred, and what still needs validation.
- Every plan should connect to a decision memo instead of reading like a generic 30/60/90 template.
- Every output should be useful to a founder operator in Mexico, not only to an investor or consultant.
- Every output should make implementation concrete: owner, timeline, dependency, and success metric where relevant.
- Penalize elegant prose without decision usefulness.
